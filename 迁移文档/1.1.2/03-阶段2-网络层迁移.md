# RTSbuilding 阶段 2 — 网络层迁移

## 状态：✅ 完成

## 日期：2026-06-07

## 目标
将所有 NeoForge `CustomPacketPayload` record + `IPayloadContext` handler 转换为 Forge 1.7.10 `IMessage` + `IMessageHandler` 模式。

## 执行记录

### 网络基础设施
- `RtsNetworkManager.java` — `SimpleNetworkWrapper` 注册中心，62 discriminators，7 个子包
- `CommonProxy.preInit()` — 调用 `RtsNetworkManager.registerMessages()`

### 已迁移子包

| 子包 | C2S | S2C | 序列化详情 |
|------|-----|-----|-----------|
| `camera/` | 2 | 1 | boolean, float×7, double×3 |
| `feedback/` | 0 | 1 | float + boolean |
| `craft/` | 5 | 2 | int, String(UTF), ItemStack (ByteBufUtils), List |
| `progression/` | 7 | 2 | int, boolean, String(UTF), List&lt;String&gt;, byte |
| `builder/` | 13 | 4 | boolean, int×3(BlockPos), ItemStack, List&lt;Integer&gt;, float×3(ray) |
| `storage/` | 18 | 2 + `RtsStorageSort.java`(enum) | int, boolean, String(UTF), ItemStack, byte(enum ordinal) |
| `blueprint/network/` | 1 | 1 | BlockPos, List&lt;BlockPos&gt;, int, String |

**总计**: 46 C2S Message + 12 S2C Message = 58 Message 类

### 关键转换

| NeoForge 1.21.1 | Forge 1.7.10 |
|-----------------|-------------|
| `record XxxPayload(...) implements CustomPacketPayload` | `class XxxMessage implements IMessage` + fields + getters + 无参构造 |
| `RegistryFriendlyByteBuf` | `ByteBuf` (io.netty.buffer) |
| `StreamCodec.of(encoder, decoder)` | `toBytes(ByteBuf)` / `fromBytes(ByteBuf)` |
| `buf.writeVarInt()` / `readVarInt()` | `buf.writeInt()` / `readInt()` |
| `buf.writeUtf(str)` | 手动: `writeInt(len) + writeBytes(bytes)` |
| `buf.writeBlockPos(pos)` | `writeInt(x); writeInt(y); writeInt(z)` |
| `ItemStack.OPTIONAL_STREAM_CODEC` | `ByteBufUtils.writeItemStack/readItemStack` |
| `IPayloadContext` | `MessageContext` |
| All handlers return `null` (桩) | 阶段 5 填充 |

### 未迁移
- `RtsClientPayloadBridge.java` — Forge 1.7.10 无此概念
- `*NetworkHandlers.java` (6 files) — 逻辑分发到各 Message Handler 类
- `*Packets.java` (6 files) — 注册集中在 RtsNetworkManager
