## 4. 软依赖策略

所有兼容层采用 `@Optional` + 反射的软依赖模式：

```java
// 示例：AE2 兼容
@Mod.Optional.Method(modid = "appliedenergistics2")
public void initAe2Compat() { ... }

// 运行时检测
if (Loader.isModLoaded("appliedenergistics2")) {
    RtsAe2Compat.init();
}
```

模组核心功能不依赖任何兼容层 — AE2/NEI/NEC 缺失时降级运行（模拟数据、子串搜索、限制功能）。

## 6. 风险与注意事项

| 风险 | 级别 | 缓解措施 |
|------|------|---------|
| GTNH AE2 API 与标准 AE2 不同 | 🔴 高 | 查阅 GTNH AE2 Unofficial 源码，使用反射/`@Optional` 软依赖 |
| NEI API 在 1.7.10 不兼容 JEI 接口 | 🟡 中 | NEI 有自己的 API（`codechicken.nei`），全新适配 |
| RtsCameraEntity 在 GTNH 中实体 ID 冲突 | 🟡 中 | 检查 GTNH 已有实体 ID，预留范围 |
| 蓝图文件格式兼容（.nbt/.schem/.litematic） | 🟡 中 | 使用 NBT 解析 + 方块映射表 |
| GUI 线程安全（世界渲染 + GUI 并发） | 🟢 低 | 1.7.10 是单线程渲染模型 |
| AE2/NEI 未安装时的降级行为 | 🟡 中 | 模拟数据 + 子串搜索 + 功能提示 |
