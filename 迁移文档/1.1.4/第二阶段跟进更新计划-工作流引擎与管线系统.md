# 第二阶段跟进更新计划 — 工作流引擎 + 管线系统

> 方向：建立核心架构，支持并发工作流和类型安全管线

**制定日期**: 2026-06-19  
**基准版本**: main v1.1.4 / port 阶段8开始  
**前置依赖**: 第一阶段 UI 设计完善（已完成）  
**预估工作量**: 5-8 天（单人全职）

---

## 一、阶段目标

移植 main 版本的两个核心架构模块：

1. **工作流引擎 (Workflow Engine)** — 支持并发操作、暂停/恢复、超时清理、持久化
2. **管线系统 (Pipeline System)** — 类型安全的操作编排，支持 8 条业务管线

这两个模块是后续所有高级功能（撤销、寻路、渲染器）的基础。

### 2.2 实现步骤

**步骤 1: 模型层 (0.5 天)**
- `RtsWorkflowType.java` — 8 种工作流类型枚举

**步骤 4: 持久化 (0.5 天)**

- `RtsWorkflowSyncService.java` — 工作流进度推送到客户端
- 配合阶段 9 的 `S2CRtsWorkflowProgressPayload` 和 `S2CRtsWorkflowProgressBatchPayload`

**步骤 6: 超时服务 (0.5 天)**

**步骤 7: 集成测试 (0.5 天)**
- 集成到 `RtsMineManager` 替换当前单线程挖掘

| 管线 | 阶段 | 完成方式 |
|------|------|----------|
| MINE_SINGLE | ProgressionGate → SessionValidate → Dimension → StopPrevious → WorkflowStart → ToolBorrow → MiningExecute → UiRefresh | asyncCompletion |
| ULTIMINE | ProgressionGate → SessionValidate → Dimension → StopPrevious → WorkflowStart → ToolBorrow → UltimineExecute → Progress → Sync → UiRefresh | tickable |
| AREA_MINE | 同 ULTIMINE | tickable |
| AREA_DESTROY | 同 ULTIMINE | tickable |
| PLACE_SINGLE | SessionValidate → WorkflowStart → PlacementExecute → UiRefresh | asyncCompletion |
| PLACE_BATCH | SessionValidate → WorkflowStart → PlacementExecute → PendingPlacement → UiRefresh | asyncCompletion |
| QUICK_BUILD | SessionValidate → WorkflowStart → PlacementExecute → UiRefresh | asyncCompletion |
| STOP_MINING | SessionValidate → StopMining | 同步 |

**步骤 10: 集成重构 (1 天)**

- 重构 `RtsStorageManager` 放置逻辑使用管线系统

## 七、验收标准

1. **工作流引擎**:
   - 支持每玩家 8 个并发工作流槽位
   - 支持创建/暂停/恢复/取消操作
   - 支持 NBT 持久化和断线恢复
   - 超时服务自动清理僵尸工作流
   - 事件总线正确分发生命周期事件

2. **管线系统**:
   - 8 条业务管线注册并可用
   - 管线执行支持同步、asyncCompletion、tickable 三种完成方式
   - 管线上下文类型安全 (TypedKey)
   - 管线结果密封 (Success/Failure/Skip)

3. **集成验证**:
   - `RtsMineManager` 使用管线系统执行挖掘
   - `RtsStorageManager` 使用管线系统执行放置
   - `./gradlew build` 编译通过
   - `./gradlew spotlessCheck` 格式检查通过
