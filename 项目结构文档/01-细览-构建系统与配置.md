## 1. 构建系统概览

本项目使用 **GTNH Convention** 构建系统，基于 **Gradle 9.3.1 + RetroFuturaGradle**。

### 核心文件

| 文件 | 作用 | 行数/大小 |
|------|------|----------|
| `build.gradle.kts` | 构建脚本（仅 3 行，其余全部自动） | 3 行 |
| `settings.gradle.kts` | 插件仓库 + GTNH Settings Convention 版本 | 20 行 |
| `gradle.properties` | 全部 GTNH 构建约定配置 | 229 行 |
| `dependencies.gradle` | 模组依赖声明 | 12 行 |
| `repositories.gradle` | 附加仓库（当前为空） | 4 行 |
| `jitpack.yml` | JitPack CI 预构步骤 | 2 行 |
| `gtnhShared/` | Spotless 代码格式化规则 | - |

## 2. build.gradle.kts

```kotlin
plugins {
    id("com.gtnewhorizons.gtnhconvention")
}
```

仅声明 GTNH Convention 插件，所有依赖管理、Mixin 处理、源码编译、打包均由此插件自动化处理。

## 3. settings.gradle.kts

```kotlin
pluginManagement {
    repositories {
        maven {
            name = "GTNH Maven"
            url = uri("https://nexus.gtnewhorizons.com/repository/public/")
            mavenContent {
                includeGroup("com.gtnewhorizons")
                includeGroupByRegex("com\\.gtnewhorizons\\..+")
            }
        }
        gradlePluginPortal()
        mavenCentral()
        mavenLocal()
    }
}

plugins {
    id("com.gtnewhorizons.gtnhsettingsconvention") version("2.0.20")
}
```

- 从 GTNH Nexus 仓库获取所有 `com.gtnewhorizons*` 依赖
- 使用 `gtnhsettingsconvention` v2.0.20 管理 Gradle 设置

## 4. gradle.properties 关键配置

### 4.1 模组基本属性

```properties
modName = RTS Building Build From Above
modId = rtsbuilding
modGroup = com.rtsbuilding.rtsbuilding
useModGroupForPublishing = true
autoUpdateBuildScript = false
```

### 4.2 Minecraft / Forge 版本

```properties
minecraftVersion = 1.7.10
forgeVersion = 10.13.4.1614
channel = stable            # MCP 渠道
mappingsVersion = 12        # MCP 映射版本
remoteMappings = https://raw.githubusercontent.com/MinecraftForge/FML/1.7.10/conf/
developmentEnvironmentUserName = Developer
```

### 4.3 Java 语法降级

```properties
enableModernJavaSyntax = jvmDowngrader
jvmDowngraderStubsProvider = shade
```

- `jvmDowngrader`：允许使用 Java 17+ 语法和标准库 API，编译时自动降级为 J8 字节码
- `shade`：将最小化的 API 存根内嵌到 jar 中（需验证许可证）

### 4.4 Mixin 配置

```properties
usesMixins = true
mixinsPackage = mixin
usesMixinDebug = false
separateMixinSourceSet = 
mixinPlugin = 
forceEnableMixins = false
containsMixinsAndOrCoreModOnly = false
```

### 4.5 源码增强

```properties
enableGenericInjection = true
```

将反编译源码中的 `List`、`Map` 等注入泛型参数（`List<E>`、`Map<K,V>`），改善开发体验。

### 4.6 版本 Token 生成

```properties
generateGradleTokenClass = com.rtsbuilding.rtsbuilding.Tags
gradleTokenVersion = VERSION
```

编译时自动生成 `Tags.java` 类，包含 `public static final String VERSION` 字段。

### 4.7 发布属性

```properties
usesMavenPublishing = true
useModGroupForPublishing = true
modrinthProjectId = 
modrinthRelations = 
curseForgeProjectId = 
curseForgeRelations = 
```

### 4.8 其他构建选项

```properties

includeWellKnownRepositories = true
usesShadowedDependencies = false
minimizeShadowedDependencies = true
```

## 5. dependencies.gradle

```groovy
dependencies {
    compileOnly("com.github.GTNewHorizons:GTNHLib:0.7.10:dev")
    compileOnly("com.github.GTNewHorizons:NotEnoughItems:2.8.48-GTNH:dev")
    compileOnly("com.github.GTNewHorizons:Applied-Energistics-2-Unofficial:rv3-beta-695-GTNH:dev")
}
```

全部采用 `compileOnly` 配置，即：
- **编译时**：可见，可使用这些 API
- **运行时**：不强制要求（软依赖），模组不可用时静默降级

| 依赖 | 用途 | 可选？ |
|------|------|--------|
| `GTNHLib:0.7.10:dev` | 提供 JVM Downgrader 存根（`List.of()`, `Map.of()` 等） | 必选 |
| `NotEnoughItems:2.8.48-GTNH:dev` | NEI 配方覆盖层集成 | 可选 |
| `Applied-Energistics-2-Unofficial:rv3-beta-695-GTNH:dev` | AE2 ME 网络存储接入 | 可选 |

## 7. 构建产物

使用标准 Gradle 构建：

```bash
./gradlew build        # 编译并打包
./gradlew runClient    # 启动客户端测试
./gradlew runServer    # 启动服务端测试
```

产物位于 `build/libs/`，文件名格式：`rtsbuilding-{version}.jar`。
