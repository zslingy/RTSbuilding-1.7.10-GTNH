## 1. 网络架构概览

RTSbuilding 使用 Forge 1.7.10 的 **SimpleNetworkWrapper** 构建网络通信层，信道名为 `"rtsbuilding"`。所有消息注册在 `RtsNetworkManager.registerMessages()` 中，按顺序分配 discriminator。

### 核心文件

| 文件 | 行数 | 职责 |
|------|------|------|
| `network/RtsNetworkManager.java` | ~320 | 网络注册中心，管理全部消息注册 |
| `network/builder/` | 17 文件 | 建造操作消息 |
| `network/storage/` | 22 文件 | 存储链接操作消息 |
| `network/progression/` | 9 文件 | 进度系统消息 |
| `network/craft/` | 7 文件 | 合成终端消息 |
| `network/camera/` | 3 文件 | 相机控制消息 |
| `network/feedback/` | 1 文件 | 伤害反馈消息 |
| `blueprint/network/` | 2 文件 | 蓝图专属网络消息 |

### 消息总量：~63 条

- **C2S (客户端→服务端)**: ~43 条
- **S2C (服务端→客户端)**: ~16 条
- **Blueprint 保留**: discriminator 60-79

## 9. 注册模式

每条消息注册使用 Forge `SimpleNetworkWrapper` 标准模式：

```java
NETWORK.registerMessage(
    {MessageClass}.Handler.class,   // IMessageHandler 实现
    {MessageClass}.class,           // IMessage 实现
    disc++,                         // discriminator (自增)
    Side.SERVER / Side.CLIENT       // 目标端
);
```

- **C2S**: 目标端 `Side.SERVER`（服务端接收并处理）
- **S2C**: 目标端 `Side.CLIENT`（客户端接收并处理）

消息均遵循 `IMessage + IMessageHandler<...>` 接口模式。

## 10. 网络数据流示意

```
客户端 GUI 操作
    │
    ▼
C2S 消息 ──── SimpleNetworkWrapper("rtsbuilding") ────▶ 服务端 Handler
                                                              │
                                                              ▼
                                                        服务端处理逻辑
                                                          (server/ 包)
                                                              │
                                                              ▼
                                                         S2C 消息
                                                              │
    ◀───────────────────────────────────────────────────────┘
    │
    ▼
客户端 ViewModel 更新 → UI 刷新
```
