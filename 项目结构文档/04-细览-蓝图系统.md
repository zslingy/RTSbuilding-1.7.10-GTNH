## 1. 蓝图系统概览

RTSbuilding 蓝图系统支持从多种格式导入建筑模板，在客户端预览后在服务端批量放置。系统包含格式解析、数据模型、坐标替换和网络通信四个子系统。

### 文件清单（14 个文件）

```
blueprint/
├── RtsBlueprint.java              # 蓝图数据模型
├── RtsBlueprintBlock.java         # 蓝图方块条目
├── BlueprintFormat.java           # 蓝图格式枚举
├── BlueprintParseException.java   # 解析异常
├── BlueprintReplaceRules.java     # 方块替换规则
├── BlueprintTransform.java        # 蓝图变换（旋转/镜像/移动）
├── format/
│   ├── BlueprintReaders.java      # 统一读取器入口
│   ├── BlueprintWriters.java      # 统一写入器入口
│   ├── BuildingGadgetsTemplateReader.java
│   ├── LitematicReader.java
│   ├── SpongeSchemReader.java
│   └── VanillaStructureNbtReader.java
└── network/
    ├── C2SBlueprintPlaceMessage.java
    └── S2CBlueprintStatusMessage.java
```

## 2. 数据模型

### 2.1 RtsBlueprint.java

核心蓝图不可变数据模型：

```java
public class RtsBlueprint {
    private final String name;                  // 蓝图名称
    private final String sourceName;            // 来源文件名
    private final BlueprintFormat format;       // 来源格式
    private final RtsBlockPos size;             // 尺寸 (x, y, z)
    private final List<RtsBlueprintBlock> blocks;       // 方块列表（不可变）
    private final Map<String, Integer> requiredItems;  // 所需材料统计
}
```

- 使用 `RtsBlueprint.create()` 工厂方法自动统计所需材料
- 提供 `getBlockCount()` 和 `getNonAirBlockCount()` 方法
- 所有集合字段均为不可变（`Collections.unmodifiableList/Map`）

### 2.2 RtsBlueprintBlock.java

```java
// 代表蓝图中的一个方块条目
属性: stateId (minecraft:stone), properties, position
支持: missingBlock 标记（未识别的方块）
```

### 2.3 BlueprintFormat.java

```java
public enum BlueprintFormat {
    LITEMATIC,           // Litematica 格式
    SPONGE_SCHEM,        // Sponge Schematic 格式
    BUILDING_GADGETS,    // Building Gadgets 模板格式
    VANILLA_STRUCTURE_NBT, // 原版结构 NBT 格式
    RTS_BLUEPRINT        // RTSbuilding 自有格式
}
```

## 3. 格式读写器

### 3.1 BlueprintReaders.java

统一读取入口，根据格式分发到对应读取器：

```java
public static RtsBlueprint read(InputStream is, String sourceName, BlueprintFormat format)
```

### 3.2 BlueprintWriters.java

统一写入入口，支持导出 RTS 蓝图到不同格式。

### 3.3 支持的导入格式

| 读取器 | 格式 | 说明 |
|--------|------|------|
| `LitematicReader` | Litematica `.litematic` | 轻量级 Minecraft 建筑格式 |
| `SpongeSchemReader` | Sponge `.schem` | 高性能 Schematic 格式 (v1/v2/v3) |
| `BuildingGadgetsTemplateReader` | Building Gadgets `.json` | JSON 模板格式 |
| `VanillaStructureNbtReader` | 原版 `.nbt` | Minecraft 原版结构 NBT |

## 4. 蓝图变换系统

### BlueprintTransform.java

支持以下变换操作：

- **旋转**: 90°/180°/270° 绕 Y 轴
- **镜像**: X 轴 / Z 轴
- **移动**: 偏移向量
- **尺寸适配**: 自动计算变换后的边界

### BlueprintReplaceRules.java

方块替换规则：

- 占位符替换（如 `stone_bricks` → `#any_stone`）
- 版本迁移映射（不同 MC 版本间的方块 ID 映射）

## 5. 蓝图网络通信

### 5.1 C2SBlueprintPlaceMessage.java

客户端 → 服务端：

- 携带完整蓝图数据 + 放置起始坐标 + 替换规则
- 触发服务端 `BlueprintPlaceService`（阶段 3 注册）

### 5.2 S2CBlueprintStatusMessage.java

服务端 → 客户端：

- 放置进度反馈（成功/失败/已放置数量/所需物品不足）
- 错误信息提示

### 独立 Discriminator 范围

蓝图消息使用 discriminator 60-79，避免与主网络消息冲突：

```java
// 在 RtsNetworkManager.registerMessages() 末尾:
// ---- Blueprint (reserved discriminators 60-79, registered in stage 3) ----
```

## 6. 蓝图工作流

```
1. 用户导入蓝图文件
   ↓
2. BlueprintReaders.read() 解析
   ↓
3. RtsBlueprint 对象创建 + 所需材料统计
   ↓
4. BlueprintPanel 预览（客户端）
   ↓
5. 用户选择放置位置
   ↓
6. C2SBlueprintPlaceMessage 发送
   ↓
7. 服务端验证（权限/材料/半径）
   ↓
8. 服务端逐一放置方块
   ↓
9. S2CBlueprintStatusMessage 反馈进度
   ↓
10. 客户端渲染动画 + 更新材料消耗
```

## 7. 配置关联

| 配置项 | 说明 |
|--------|------|
| `enableBlueprints` (`Config.areBlueprintsEnabled()`) | 启用/禁用整个蓝图功能 |
| `maxBlueprintBlocks` (`Config.maxBlueprintBlocks()`) | 单个蓝图最大非空气方块数（1-200000，默认 20000） |

当 `enableBlueprints = false` 时，蓝图面板和导入功能从 UI 中隐藏，网络消息处理返回错误。
