## 1. 客户端架构概览

RTSbuilding 客户端采用 **MVP (Model-View-Presenter)** 变体架构：

- **Model**: 5 个 ViewModel（通过 `RtsClientState` 单例统一管理）
- **View**: 10 个 Panel + 4 个 Screen + 3 个 Overlay
- **Input**: `RtsInputRouter` 统一路由鼠标/键盘/滚轮事件
- **Renderer**: 6 个世界渲染器注册在 Forge 事件总线上

## 2. RtsScreen — 主 GUI 屏幕

### 2.1 核心职责

替代原 1.21.1 版 2426 行的 `BuilderScreen`，采用 **单层容器设计**：

```java
public class RtsScreen extends GuiScreen {
    private final RtsClientState state;
    private final RtsInputRouter inputRouter;
    private final Map<String, IRtsPanel> panels;   // 有序面板集合
}
```

### 2.2 面板注册 (initGui)

```java
addPanel(new RtsTopBarPanel());       // 顶部工具栏
addPanel(new StorageGridView());      // 存储网格
addPanel(new StorageCategoryView());  // 分类侧栏
addPanel(new RecentGridView());       // 最近使用
addPanel(new CraftPanelView());       // 合成面板
addPanel(new CraftQuantityDialog());  // 数量选择弹窗
addPanel(new CraftFeedbackPopup());   // 合成反馈弹窗
addPanel(new PinSlotView());          // 工具槽
addPanel(new BlueprintPanel());       // 蓝图面板
```

### 2.3 初始数据请求

进入屏幕时自动发起 3 个 C2S 请求：

```java
RtsNetworkManager.NETWORK.sendToServer(new C2SRtsRequestStoragePageMessage(0, 0));
RtsNetworkManager.NETWORK.sendToServer(new C2SRtsRequestCraftablesMessage("", false, 0, 50, false, null));
RtsNetworkManager.NETWORK.sendToServer(new C2SRtsRequestProgressionStateMessage());
```

### 2.4 渲染流程

```java
drawScreen(mouseX, mouseY, partialTicks):
  1. drawDefaultBackground()
  2. state.resetAllFrameStates()
  3. GL11.glScalef(zoom, zoom, 1.0f)       // UI 缩放
  4. 遍历 panels → panel.render(this, scaledX, scaledY, partialTicks)
  5. GL11.glPopMatrix()
  6. 绘制底部状态提示
```

### 2.5 输入分发

| 事件 | 方法 | 路由 |
|------|------|------|
| 鼠标点击 | `mouseClicked()` | `inputRouter.dispatchClick()` |
| 鼠标滚轮 | `handleMouseInput()` | `inputRouter.dispatchScroll()` |
| 按键 | `keyTyped()` | `inputRouter.dispatchKey()` |

## 3. ViewModel 系统 (RtsClientState)

`RtsClientState` 作为单例管理 5 个 ViewModel，通过 S2C 消息处理更新状态，驱动 UI 重绘。

| ViewModel | 管理数据 | 更新源 |
|-----------|---------|--------|
| `CameraViewModel` | 相机位置、缩放、模式 | `S2CRtsCameraStateMessage` |
| `StorageViewModel` | 存储条目、分页、排序、搜索 | `S2CRtsStoragePageMessage` |
| `CraftViewModel` | 可合成列表、合成状态 | `S2CRtsCraftablesMessage`, `S2CRtsCraftFeedbackMessage` |
| `InteractionViewModel` | 鼠标悬停目标、交互模式 | 本地输入状态 |
| `ProgressionViewModel` | 解锁节点、进度树 | `S2CRtsProgressionStateMessage` |

## 4. IRtsPanel 面板协议

```java
public interface IRtsPanel {
    boolean isVisible();                          // 是否可见
    void render(RtsScreen screen, int mx, int my, float pt);  // 渲染
    boolean handleClick(int mx, int my, int btn);  // 点击处理
    boolean handleScroll(int mx, int my, int dir); // 滚轮处理
    boolean handleKey(char c, int code);           // 按键处理
    void resetFrameState();                        // 重置帧状态
}
```

所有面板通过此协议实现，由 `RtsScreen` 和 `RtsInputRouter` 统一调度。

## 5. RtsInputRouter — 输入路由

```
点击/按键/滚轮事件
    ↓
RtsInputRouter
    ↓ 遍历 panels（按优先级）
第一个返回 true 的面板 → 消费事件
全部返回 false → 传递给 super 方法
```

支持：
- 面板优先级排序
- 模态弹窗拦截（如 `CraftQuantityDialog`）
- 帧级状态重置

## 6. 世界渲染器 (render/)

通过 `RtsWorldRenderer` 单例注册到 `MinecraftForge.EVENT_BUS`，非俯视视角时不渲染。

| 渲染器 | 功能 | 关键特性 |
|--------|------|---------|
| `RtsWorldRenderer` | 总控，调度所有子渲染器 | 注册在 EVENT_BUS，管理 GL 状态 |
| `ShapeGhostRenderer` | 方块放置预览（半透明） | 线框模式/实体模式切换 |
| `BoundaryRenderer` | 操作半径边界线 | 根据技能半径动态调整 |
| `InteractionTargetRenderer` | 鼠标指向方块高亮 | 支持面选择高亮 |
| `AnimationRenderer` | 放置/破坏动画 | 缩放 + 淡入淡出 |
| `StorageHighlightRenderer` | 链接存储容器高亮 | 区分已链接/未链接 |

## 9. 快速建造面板 (QuickBuildView)

支持 7 种形状的快速建造：

| 形状 | 纹理 | 说明 |
|------|------|------|
| 单方块 | `single_block.png` | 放置一个方块 |
| 线 | `line_block.png` | 沿方向放置一行 |
| 方形 | `square_block.png` | 平面矩形 |
| 立方体 | `box_block.png` | 3D 立方体 |
| 圆形 | `circle_block.png` | 平面圆形 |
| 圆环 | `radio_button.png` | 中空环形 |
| 墙 | `wall_block.png` | 垂直墙面 |

### 操作流程

1. 选择形状 → 2. 选择方块类型 → 3. 设置尺寸（拖拽/输入）→ 4. 预览 `ShapeGhostRenderer` → 5. 确认 → 6. `C2SRtsPlaceBatchMessage` 发送

## 10. 客户端数据流

```
用户操作
    │
    ├─→ 本地输入（相机移动、形状选择）
    │       ↓
    │   RtsInputRouter → RtsClientState (ViewModel)
    │       ↓
    │   本地渲染更新（无需网络）
    │
    └─→ 需服务端确认的操作（放置/破坏/存储/合成/进度）
            ↓
        C2S 消息发送
            ↓
        S2C 消息返回 → ViewModel.update() → UI 刷新
```
