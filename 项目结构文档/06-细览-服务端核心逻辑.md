## 2. RtsStorageManager — 存储总控

服务端最核心的类（原 2416 行），统一管理：

| 职责 | 说明 |
|------|------|
| 存储链接 | 玩家链接/解除箱子等容器的管理 |
| 物品检索 | 从所有已链接容器汇总物品列表 |
| 存取操作 | 取出/存入/快速移动物品 |
| 排序 | 页面排序（名称/数量/ID） |
| 自动存储 | 挖矿自动存入链接容器 |
| 漏斗模式 | 将指定物品自动漏斗到目标容器 |
| AE2 集成 | ME 网络物品种类查询和存取 |
| 远程菜单 | 远程打开容器 GUI + 操作 |

### 关键数据流

```
C2S 存储消息
    ↓
RtsStorageManager (处理逻辑)
    ├─→ RtsStorageSessionStore (获取玩家会话)
    ├─→ RtsStorageSession (当前玩家存储视图)
    │       ├─→ 普通容器扫描
    │       ├─→ AE2 ME 网络查询 (via RtsAe2Compat)
    │       └─→ 聚合/排序/分页
    ├─→ 物品操作（取出/存入/传送）
    └─→ S2C 响应消息
```

## 4. 子包详解

### 4.1 camera/ — 相机管理

`RtsCameraManager.java`：

- 管理服务端授权相机状态
- 验证相机移动请求（边界/权限/技能半径）
- 生成 `RtsCameraEntity` 实体
- 处理相机同步

### 4.2 data/ — 持久化数据

| 文件 | 用途 |
|------|------|
| `PlacedBlockTrackerData.java` | 追踪 RTS 放置的方块（用于回滚/统计） |
| `RtsSharedProgressionData.java` | 团队共享进度数据 |
| `RtsStorageSessionStore.java` | 玩家存储会话持久化存储 |

### 4.3 feedback/ — 伤害反馈

`RtsDamageFeedbackManager.java`：

- 管理服务器端伤害计算
- 向客户端发送 `S2CRtsDamageFeedbackMessage`

### 4.4 loadout/ — 采矿装备配置

| 文件 | 用途 |
|------|------|
| `MiningLoadoutRole.java` | 采矿工具角色枚举（镐/斧/铲等） |
| `MiningLoadoutState.java` | 当前装备状态管理 |
| `RtsMiningRules.java` | 采矿规则（工具匹配、挖掘速度计算） |

### 4.5 menu/ — 容器菜单

`RtsCraftTerminalMenu.java`：

- AE2 合成终端对应的服务端容器
- 管理合成槽位和物品传送

### 4.6 policy/ — 破坏策略

`RtsBreakPolicy.java`：

- 验证方块破坏权限
- 检查技能树解锁状态
- 应用工具耐久消耗

### 4.7 progression/ — 进度管理

`RtsProgressionManager.java`（详见 [细览 07](07-细览-技能树与进度系统.md)）：

- 管理玩家技能节点解锁
- 验证解锁请求（消耗材料检查）
- 应用解锁效果
- 团队共享进度同步

### 4.8 storage/ — 存储子系统

| 文件 | 用途 |
|------|------|
| `GuiBinding.java` | GUI 与容器坐标绑定 |
| `OverflowOutcome.java` | 物品溢出处理结果枚举 |
| `RecipeInputExtractor.java` | 从配方提取输入物品列表 |
| `RecipeScanCache.java` | 配方扫描缓存（惰性重建） |
| `RtsStorageSession.java` | 单个玩家存储会话（见 §3） |

### 4.9 tracking/ — 方块追踪

`RtsBlockTrackingEvents.java`：

- 监听方块放置/破坏事件
- 更新放置追踪数据
- 触发相关反馈

## 5. 网络消息处理链

```
C2S Message → Handler.onMessage()
    ├─ Builder handlers     → 世界操作（放置/破坏/采矿/批量建造）
    ├─ Camera handlers      → RtsCameraManager
    ├─ Craft handlers       → RtsCraftTerminalMenu / RecipeScanCache
    ├─ Progression handlers → RtsProgressionManager
    └─ Storage handlers     → RtsStorageManager → RtsStorageSession
         ↓
     验证通过 → 执行操作 → S2C 响应
     验证失败 → S2C 错误消息
```

## 6. 安全验证层次

每一次 C2S 请求在服务端至少经过以下验证：

| 层次 | 检查项 |
|------|--------|
| 玩家身份 | 消息中的 `EntityPlayerMP` 校验 |
| 权限 | OP / 创造模式 / 生存模式差异 |
| 技能 | 对应技能节点是否解锁 |
| 半径 | 操作目标是否在允许半径内 |
| 材料 | 背包/存储中是否有足够材料 |
| 世界保护 | 目标方块是否可破坏/可放置 |

## 7. 配方缓存（RecipeScanCache）

`RecipeScanCache.java` 是性能关键组件：

- 惰性加载：首次合成请求时扫描全部存储中的配方
- `markDirty()`：在 `CommonProxy.serverStarting()` 调用，标记服务端重启后需重建
- 缓存过滤：按"可合成"过滤所有配方，仅返回存储中有足够材料的配方
- `RecipeInputExtractor` 辅助提取配方输入列表
