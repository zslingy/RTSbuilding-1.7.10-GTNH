## 1. 系统概览

RTSbuilding 的生存模式技能树定义了 18 个渐进式解锁节点，玩家需要通过消耗物品来解锁高级功能。系统分为两部分：

- **数据定义层** (`progression/`)：定义节点、消耗、效果
- **服务端管理层** (`server/progression/`)：管理玩家解锁状态、验证请求、应用效果

### 文件清单

```
progression/                      ← 数据定义
├── RtsProgressionNodes.java     ← 18 个节点定义
├── RtsProgressionNode.java      ← 节点数据模型
├── RtsFeature.java              ← 功能枚举
├── RtsIngredientCost.java       ← 消耗项
└── RtsUnlockEffect.java         ← 解锁效果

server/progression/              ← 服务端管理
└── RtsProgressionManager.java   ← 进度管理器

server/data/
└── RtsSharedProgressionData.java ← 团队共享数据

client/
├── ProgressionViewModel.java     ← 客户端进度 ViewModel
└── screen/RtsProgressionScreen.java ← 技能树 GUI
```

## 6. 消耗覆盖系统

### 6.1 Config 层

```properties
progressionCostOverrides = [
    "ultimine=minecraft:diamond_pickaxe:1,minecraft:redstone_block:1"
]
```

### 6.2 运行时 API

```java
// 设置特定节点消耗覆盖
Config.setProgressionCostOverride("ultimine", "minecraft:diamond_pickaxe:1,minecraft:redstone_block:1");

// 网络同步覆盖
C2SRtsSetProgressionCostMessage → RtsProgressionNodes.applySyncedCostOverrides(...)
```

服务器管理员可通过 GUI 或命令修改任意节点的解锁消耗，修改后立即生效（`config.save()`）。

## 7. 进度管理器（RtsProgressionManager）

```java
// 核心方法
boolean unlockNode(EntityPlayerMP player, ResourceLocation nodeId)
    → 验证前置节点已解锁
    → 验证材料消耗
    → 扣除物品
    → 应用解锁效果
    → 同步状态到客户端

boolean hasNodeUnlocked(EntityPlayer player, ResourceLocation nodeId)
boolean hasFeatureUnlocked(EntityPlayer player, RtsFeature feature)
int getActiveRadius(EntityPlayer player)
```

### 解锁流程

```
C2SRtsUnlockProgressionNodeMessage
    ↓
RtsProgressionManager.unlockNode(player, nodeId)
    ├─ 验证 nodeId 是否存在
    ├─ 验证前置节点已解锁
    ├─ 验证 material 检查（从链接存储扣减）
    ├─ 标记节点为已解锁
    ├─ 应用 RtsUnlockEffect
    │     ├─ unlock(RtsFeature) → 添加功能权限
    │     └─ radius(int)        → 更新操作半径
    ├─ 持久化解锁状态
    └─ S2CRtsProgressionStateMessage 同步客户端
```

## 8. 团队共享

当 `shareSurvivalProgressionWithTeams = true` 时：

- 解锁状态同步给同队伍玩家
- 家锚点 (`RtsHome`) 团队内共享
- 通过 `RtsSharedProgressionData` 管理

---

## 9. 客户端 ViewModel

`ProgressionViewModel` 接收 `S2CRtsProgressionStateMessage` 后更新：

- `unlockedNodes`: 已解锁的节点 ID 集合
- `availableNodes`: 当前可解锁的节点（前置已满足）
- `activeRadius`: 当前有效操作半径

`RtsProgressionScreen` 以树形/网格布局显示技能树，点击可解锁节点触发 `C2SRtsUnlockProgressionNodeMessage`。

## 10. 配置开关

| 配置项 | 说明 |
|--------|------|
| `enableSurvivalProgression` | **false** = 所有功能直接可用，**true** = 需按技能树解锁 |
| `shareSurvivalProgressionWithTeams` | 是否团队共享进度 |
| `progressionCostOverrides` | 节点消耗覆盖（管理员可修改） |

当 `enableSurvivalProgression = false` 时，`maxActionRadiusBlocks` 直接作为操作半径使用；所有网络消息中的技能验证返回 `true`。
